⏺ 分析

  Call Stack を見ると、ユーザーコードは一切見えず 全て Next.js / React 
  の内部（react-server-dom-turbopack-client、layout-router）だけです。

  これはサーバーコンポーネントが例外を投げ → RSC 
  ペイロードとしてクライアントに送られてエラー画面になったパターンです。実際のエ
  ラーメッセージはターミナルにあります。

  原因の特定

  犯人はほぼ確実に vm モジュールです：

  Next.js 15 + Turbopack は、サーバーコンポーネントを
  Node.js の通常モジュールとは異なる形でバンドルする。
  vm モジュール（動的コード実行）は Turbopack の
  サンドボックス環境で利用できない or 制限される。

  対処方針

  vm を排除して、正規表現で data.js をパースする方法に切り替えます。

  data.js の各行は以下の形式で規則的なので、正規表現で安全に解析できます：

  Question["Head"][0] = ["問題文", "選択肢1", "選択肢2", "選択肢3", "選択肢4",
  1];

  - fs と path だけ使用（Node.js 標準、Turbopack でも安定）
  - vm / eval / new Function は一切使わない
  - 動作が予測可能でシンプル

  この方針で page.tsx を修正してよいですか？
